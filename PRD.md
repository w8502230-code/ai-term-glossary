# AI 术语普及 — 产品需求文档（PRD）

## 1. 背景与目标

### 1.1 背景

生成式 AI 相关概念更新快、术语多，非 IT 背景用户难以理解专业表述，需要一种**低门槛、可反复使用**的轻量工具做普及。

### 1.2 产品定位

一款**独立**的 Web 应用（手机浏览器与桌面浏览器均可通过链接访问），以「查一个词 → 看懂两段话」为核心，帮助用户建立对 AI 术语的基础认知。

### 1.3 成功标准（MVP）

- 用户能在首页通过**文字、语音**或点选推荐卡片提交术语，进入**同一页**查看「专业定义（先中文、再英文）+ 白话解释（仅中文）」。
- 用户能选择「继续」回到首页查下一个词，或「结束」离开流程。
- 首页每次返回时展示**不同的 3 张推荐术语卡片**（来自本地词池洗牌，不依赖额外 AI 调用）。
- 交互简洁、动效克制；**不在前端暴露 AI API Key**。
- **互动与曝光**：在**首页**与**解释页**（用户完成查询后展示释义的那一页；若后续实现卡片翻转动效，互动条放在与释义同一视觉层级即可）展示 **点赞、转发（符号）** 与 **浏览（`views` 文案 + 数字）**，规则见 §5.6（实现可与「解释页」主流程分阶段交付，但文档与验收以此为准）。

### 1.4 分发、分享与是否需要部署

- 本应用设计为**可通过链接转发给他人**（例如微信里打开、手机浏览器收藏网址）。
- **仅在你自己电脑上的 `localhost` 运行时，外人无法访问**；对方没有与你同一局域网或未做穿透时，也打不开你的链接。
- 因此：**要让「随便一个人点开就能用」**，一般需要 **部署到公网可访问的环境**（例如：前端静态托管 + 后端 API 托管在同一域名或配置好 CORS），得到 **HTTPS 网址** 再分享。
- **演示/内测**阶段若暂不上线，可选用平台提供的**临时隧道**（如 ngrok、cloudflared 等）把本地服务暴露为公网 URL，但稳定性和安全策略需自行评估，**正式转发建议走正规部署**。
- 具体部署形态与成本见 `tech-design.md` 与后续 `README.md`（实现阶段补充）。

## 2. 目标用户与场景

### 2.1 目标用户

- 无 IT 背景，但对 AI **技术、现状、趋势**有好奇心的大众用户。
- 可能通过手机分享链接进入。

### 2.2 典型场景

- 看到新闻里的「RAG」「对齐」「Token」等词，想快速弄清含义。
- 碎片化时间打开链接，连续查 2～3 个词后离开。

## 3. 范围说明

### 3.1 MVP 范围内

- 极简首页（参考 Google 首页式留白与居中搜索区）。
- **术语输入**：用户可使用**中文或英文**在输入框内键入（或混合常见写法，如「RAG」「大模型」），**回车提交**；亦可使用**语音输入**（见 §5.1）；以及**三张可点击推荐卡片**。
- **大语言模型（已定）**：生成解释统一通过 **阿里云百炼**（大模型服务平台，控制台：<https://bailian.console.aliyun.com/>）调用 **通义千问** 等模型；具体 API 形态以百炼官方文档及 `tech-design.md` 为准。产品方**已在百炼控制台具备 API Key**，由后端经环境变量配置，**不下发前端**。
- 结果页（同一页内）：
  - **专业定义**：**先用中文**撰写一段专业表述；**再用英文**撰写一段对同一术语的专业表述（顺序固定：中 → 英）。
  - **通俗解释**：**仅中文**，面向无 IT 背景读者的白话说明。
- 操作：**继续**（回首页）、**结束**（进入结束说明页）。
- 轻量 Hover / 页面过渡动画（不喧宾夺主）。
- 后端经 **百炼** 调用 **通义千问**（或文档指定的等价模型）生成上述内容；须满足在中国大陆网络环境下**可稳定访问、不依赖个人 VPN**。**不使用 DeepSeek**。

### 3.2 MVP 范围外（可后续迭代）

- 用户账号、收藏、历史记录云端同步。
- 多语言整站切换（除中文外的完整 i18n）。说明：MVP 已在解释页提供**专业定义英文段**，不属于整站多语言产品化范围。
- 社区贡献词条、人工审核后台。
- 原生 App 上架（MVP 以响应式 Web 为主）。
- **全站精确社交图谱、防刷审计、登录后点赞**：若与 §5.6 的「轻量计数」冲突，放到后续迭代。

## 4. 用户旅程与页面

### 4.1 信息架构

| 页面       | 路径（示例）     | 说明 |
|------------|------------------|------|
| 首页       | `/`              | 标题、输入框（含右侧**语音/话筒**入口）、三张推荐卡片；**互动条**（点赞 / 转发 / 浏览，见 §5.6） |
| 解释页     | `/explain?term=` | 展示专业（中→英）+ 白话（中文）；**互动条**（与首页同套视觉规范：点赞/转发符号 + `views` 文案；**点赞、转发数字与首页一致**（全站聚合），**浏览（views）按当前词条**）；继续 / 结束 |
| 结束页     | `/goodbye`       | 感谢与说明；Web 无法强制关标签，可提供返回首页入口 |

### 4.2 关键流程

1. 用户在首页**用中文或英文键入**术语并回车、或使用**语音输入**将识别结果填入输入框后提交（或识别完成后自动提交，具体交互以 §5.1 为准），或点击某张推荐卡片 → 跳转解释页（带 `term`，保留用户原始写法）。
2. 解释页加载时请求后端生成（或展示加载态）**同一页内依次展示**：**专业定义（中文段落）** → **专业定义（英文段落）** → **通俗解释（仅中文段落）**。
3. 用户点「继续」→ 回首页（推荐卡片重新洗牌）。
4. 用户点「结束」→ 结束页；可选「返回首页」。

## 5. 功能需求明细

### 5.1 首页

- **输入框**：占位文案提示「输入想了解的 AI 术语（中/英均可）」；支持回车提交。
- **语音输入（话筒）**：
  - 在输入框**右侧**提供**话筒图标按钮**，交互参考 **Google 首页搜索框右侧小话筒**（简约、可识别为「语音」）。
  - 点击后：在浏览器支持的前提下，请求**麦克风权限**；进入**聆听/录音中**状态（可用简单动画或文案提示，如「请说出术语」）；将识别到的文字**写入同一输入框**，用户可再编辑后回车，或产品可约定「识别结束自动提交」（二选一须在实现与验收中一致，**推荐：写入输入框后由用户确认回车**，以降低误触跳转）。
  - **实现约束**：优先使用浏览器 **Web Speech API**（`SpeechRecognition`）等**端侧语音识别**，不经过自有后端转写，**避免**为语音另增密钥与流量成本；若当前环境不支持或用户拒绝权限，**静默降级为仅文字输入**，并给出简短提示（如「当前浏览器不支持语音」或「请在设置中允许麦克风」）。
  - **HTTPS**：语音能力在多数浏览器中要求 **HTTPS** 或 `localhost`；与 §1.4 公网部署一致，正式分享链接需安全上下文。
- **输入语言**：接受**中文、英文**或常见混合写法（如「LLM」「大模型」）；语音以用户实际说的语言识别结果为准。后端**严格按 IT / 人工智能 / 机器学习 / 软件开发语境**理解与生成（禁止用物理、日常生活等其它领域义项替代，如「Temperature」须按大模型**采样温度**解释而非热力学温度），不因输入方式或语言不同而缺少中/英专业段或中文白话段。
- **校验**：空内容不跳转；过长输入需截断或提示（具体上限见技术设计）。
- **推荐卡片**：固定 3 张；文案为术语名（可加极短副标题，可选）。
- **动态性**：每次进入首页（含从解释页「继续」返回）重新从本地词池随机抽取 3 个术语展示。

### 5.2 解释页

- **标题区**：显示当前查询的术语（与用户输入或卡片一致）。
- **内容区 1a — 专业定义（中文）**：偏教材/百科风格的简短中文定义（控制篇幅，避免长篇论文）。
- **内容区 1b — 专业定义（英文）**：与 1a **同一术语、同一专业粒度**的英文定义；置于中文段之后，**顺序固定**。
- **内容区 2 — 通俗解释（仅中文）**：无技术前提的比喻或生活化说明；**不要求**提供英文版白话。
- **版式**：1a、1b、内容区 2 **同页纵向排列**，小标题或分区样式需让用户一眼区分「专业 / 通俗」与「中文 / 英文」。
- **按钮**：「继续」「结束」；触控区域足够大（移动端友好）。

### 5.3 结束页

- 简短致谢与「学习结束」说明。
- 提示用户可关闭标签页；提供「返回首页」弱入口（产品可接受）。

### 5.4 动效与反馈

- 卡片与按钮：Hover / Active 时有轻微缩放或阴影变化（桌面）；过渡使用 CSS transition。
- 页面切换：淡入或轻微上移入场（实现方式见技术设计）。

### 5.5 异常与降级

- 网络失败、模型超时：展示友好错误文案，允许重试或返回首页。
- 不向前端泄露 API Key 或内部堆栈细节。

### 5.6 点赞、转发、浏览（符号 + `views` 文案 + 计数）

**目标**：在 **首页（`HomePage`）** 与 **解释页（`ExplainPage`，即查词后的结果页；与 PRD 中「翻转卡片」若存在，指用户看到释义内容时的同一页）** 展示互动能力并显示计数，增强可感知热度与分享动机。

**视觉规范（产品一致）**

- **点赞、转发**：使用**符号化按钮**（非长文案按钮），**视觉参照微信小视频（短视频）播放页底部**那一排的 **点赞**与 **转发** 图标样式——即用户熟悉的「空心/实心爱心（或拇指）+ 转发箭头」一类图形语言；具体矢量图可用开源图标库中**语义等价**的图标实现，但排列与辨识度应向该参照对齐。
- **浏览**：**不使用图形图标**，固定展示英文 **`views`**（小写）作为标签，后接**数字**（如 `views 128`）；与点赞、转发排在同一互动条内即可。

**语义与交互（与视觉无关的规则）**

| 类型 | 含义 | 交互要点 |
|------|------|----------|
| **浏览** | 页面或词条被打开/阅读的次数 | **解释页**：宜在释义**成功展示后**计一次「该 `term` 的浏览」（避免仅打开页而接口失败仍涨浏览）。同一用户多次打开**同一 `term`** 的解释页是否每次都 +1，由 `tech-design.md` 约定（MVP 可每次成功展示都 +1）。**首页**：计「首页被打开」时须排除**从解释页点击「继续」返回首页**的这一类跳转——即：用户路径为「进入首页 → 查询 → 解释页 → 继续 → 再回到首页」时，**后面这一次回到首页不要新增首页浏览次数**；**首次进入应用/直接打开 `/`（含刷新、外链进入）**仍照常计。避免同一会话里反复查词把首页 PV 撑高。 |
| **点赞** | 用户对该应用的正向反馈（全站） | 点击后切换「已赞 / 未赞」态；**数字**为**全站聚合**，首页与解释页**显示同一数值**；同一浏览器内**只计一次有效点赞**（两页共用本地状态，避免首页点一次、解释页再点一次导致 +2）。 |
| **转发** | 用户触发分享传播 | 优先 **Web Share API**（移动端正向）；不支持时降级为**复制当前页链接**（含 `term` 的 explain URL 或首页 URL）。**计数**：全站聚合，首页与解释页**显示同一数值**；同一浏览器会话内**成功转发只 +1 次**（两页都完成转发仍计 1 次，见技术设计）。 |

**文案与布局**

- 点赞、转发为**符号 + 数字**；浏览为 **`views` + 数字**。小屏可单行横向排列或允许轻微滚动，**触控区域 ≥ 44px**（与 §5 其它条目一致）。
- **点赞 / 转发** 须有 **aria-label**（如「点赞」「转发」）满足读屏，**浏览**可用 `views` 文案本身或附加 `aria-label` 说明「浏览次数」。
- **解释页**的计数默认与**当前 URL 中的 `term`** 关联；**首页**若展示站点级数据，需在界面微文案上区分（如「本站」），避免用户误以为针对某一词条。

**非目标（本阶段不做）**

- 不在此需求中要求评论区、弹幕或用户主页。
- 不要求与第三方社交平台 OAuth 打通；「转发」以系统分享/复制链接为主。

## 6. 非功能需求

- **性能**：首屏可交互；解释页在合理时间内给出结果（含加载状态）。
- **安全**：**百炼 API Key 仅服务端**；对用户输入做基本长度与注入防护（见技术设计）。语音仅在用户授权下访问麦克风，**录音内容不上传自有服务器**（若采用浏览器内置识别）。
- **兼容**：主流移动浏览器与桌面浏览器；viewport 适配小屏。**语音输入**在不同浏览器/系统上能力差异较大（如 iOS Safari 限制），须符合 §5.1 的降级策略。
- **合规**：生成内容需符合平台与供应商政策；不引导违法或有害用途。

## 8. 验收清单（手动）

- [ ] 手机与桌面打开首页布局正常，输入框、**话筒按钮**与推荐卡片易于点击。
- [ ] 在支持的环境下：点击话筒、授权麦克风后，识别结果进入输入框并可成功发起查询；不支持或拒绝权限时，有合理提示且**仍可仅用文字**完成流程。
- [ ] 使用**中文术语**与**英文术语**分别查询，均能进入解释页且结构正确。
- [ ] 解释页同页展示：**专业定义（中文）** → **专业定义（英文）** → **通俗解释（仅中文）**，顺序与分区清晰。
- [ ] 回车与点击卡片均能进入解释页并显示上述全部内容。
- [ ] 「继续」回首页后，三张卡片与上次进入时不完全相同（词池足够大时多次验证）。
- [ ] 「结束」进入结束页，说明清晰。
- [ ] 浏览器网络面板中**看不到** AI 供应商 Key。
- [ ] **百炼（通义千问）** API 在**目标用户典型网络环境**下可访问（不依赖个人 VPN）。
- [ ] 错误场景有提示，应用不白屏。
- [ ] 首页与解释页均可见互动条：**点赞、转发**为类微信小视频下方的**符号**，**浏览**为 **`views` 文字 + 数字**；解释页在释义成功加载后浏览计数行为符合 §5.6。
- [ ] 点赞可点击切换状态（或符合技术设计中的等价交互）；转发在支持的环境下可调起系统分享，否则复制链接可用。
